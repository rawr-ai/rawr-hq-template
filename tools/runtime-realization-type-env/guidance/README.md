# Runtime Lab Guidance Map

This directory is the reusable operator-guidance surface for the contained
runtime-realization lab. It holds instructions, guardrails, workflows, and
record overlays that are meant to be reused across phases.

## Routing Map

| Need | Open | Use |
| --- | --- | --- |
| Proof categories, review categories, and test-theater rules | `guardrails-design.md` | Decide how evidence may be claimed and what must be downgraded |
| Lab source/test/scenario ownership | `guardrails-lab-plane-topology.md` | Route work across shared source, Oracle, Reference Runtime, and scenario packs |
| Runtime proof workflow overlay | `workflow-phased-agent-verification.md` | Apply lab-specific proof, evidence, gate, and migration review lanes to Workstream Plugin Pack workstreams |
| Runtime workstream record overlay | `workstream-record-overlay.md` | Apply lab-specific authority, evidence, gate, and fill guidance to the Workstream Plugin Pack record asset |

Guidance is not proof/status evidence. Proof ledgers and status maps live under
`../evidence/`; phase anchors and produced phase artifacts live under
`../phases/<phase>/`.
