# Architecture Inventory Router

## Scope

- Applies to the curated JSON inventory in `tools/architecture-inventory/**`.

## Boundaries

- Owns expected project config paths, tags, and named targets consumed by the
  repository `sync:check` verifier.
- Inventory entries describe declared repository facts. They do not authorize
  architecture, infer projects, or replace Nx project configuration and
  Habitat structural policy.
- Each project may appear in only one inventory file, and every `config` path
  must resolve to the project metadata that owns the compared tags and targets.
- `slice-0-first-cohort.json` is generated by the registered Nx sync generator;
  change its generator rather than hand-editing the generated output.

## Flow

- The Nx sync generator materializes the first-cohort inventory.
- The repository verifier reads every inventory file, rejects duplicate project
  declarations, and compares each entry with its owning project configuration.
