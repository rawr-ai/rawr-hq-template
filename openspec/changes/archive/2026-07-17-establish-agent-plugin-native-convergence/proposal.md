## Why

Immutable curated releases are still migration evidence until a governed accepted set can be promoted and converged through provider-native state with one truthful owner per selected home. C3 adds that inert Template-owned application layer without exposing public commands, coupling Template to personal executable code, or allowing receipts and test results to authorize themselves.

## What Changes

- Add a closed parser for mutually exclusive targeted-test, complete-test, and canonical modes, with canonical inputs resolved only from a policy-enumerated fixed channel plus explicit provider homes and a read-only content-record Git locator.
- Add deterministic provider projections and semantic capability profiles derived only from immutable release artifacts.
- Add provider-owned canonical mechanical evaluation evidence and immutable publication in the stable artifact home, plus promotion-owned governed `AcceptanceRequest` and `AcceptanceEvidence` validation. Test and deployment actors cannot issue an accepted outcome.
- Add release-input-equivalence promotion attestations and a fixed `current-main` resolver that preserves accepted artifact provenance across rewritten Git commits without rebuilding.
- Add target-scoped native provider inventory, plan, apply, visible-state verification, receipt settlement, managed retirement, status, partial outcomes, and read-only convergence.
- Integrate provider-owned inverse actions with the controller-owned bounded last-operation capsule; no provider service gains another undo store.
- Support multiple explicitly selected homes while preventing one target receipt, observation, or outcome from authorizing another.
- Keep all new applications unreachable from command discovery until C5 atomically activates `rawr agent plugins ...` and deletes the mixed aggregate.

## Capabilities

### New Capabilities

- `agent-plugin-lifecycle-mode-selection`: Closed targeted, complete, and canonical desired-state input modes and pre-side-effect validation.
- `agent-plugin-promotion`: Mechanical evidence, governed acceptance validation, release-input-equivalence promotion, and fixed current-main channel resolution.
- `agent-provider-projection`: Deterministic artifact-backed native projections and semantic provider capability compatibility.
- `agent-provider-deployment`: Live native inventory and convergence, target receipts, managed retirement, status, truthful partial outcomes, and multi-home isolation.

### Modified Capabilities

- `agent-plugin-build-artifact-store`: Store and retrieve opaque canonical mechanical-evidence bytes immutably under the existing sole Template artifact home without granting build or acceptance authority.
- `agent-plugin-undo-capsule`: Admit provider-owned target/inverse protocols into the existing controller capsule without creating provider-local undo authority.

## Impact

- Adds Template-owned promotion and provider-deployment packages/services, provider adapter contracts, stable target-receipt persistence below the controller runtime data root, a narrow build-owned mechanical-evidence store adapter, generated provider fixtures, Nx projects, and internal application tests.
- Extends only the controller capsule's closed owner protocol registry and replay path needed for provider inverse actions.
- Mines renderer and native adapter behavior from `services/agent-config-sync` and `packages/agent-config-sync-node` as migration evidence; it does not wrap, rename, import, or activate those aggregate owners. Their semantic deletion remains C5.
- Adds no command file below `apps/cli/src/commands/agent/plugins`, controller manifest entry, Oclif mutation, personal executable code, cross-repository ancestry, provider fallback, Cowork adapter, or protected-lane mutation.
