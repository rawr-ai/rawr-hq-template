# Service

`service` is the reusable contract-first oRPC service kind. Its packets apply
to standalone services and to service interiors embedded in API server
plugins. They own the stable spine and native contract, implementation,
middleware, context, module, and router relationships without freezing product
inventory or optional semantic interiors. Contract authority separately owns
the declarative module-contract grammar, direct TypeBox adaptation, and private
attached Effect errors. Error authority owns their import boundary and the
literal spelling of governed import identities. Exact vendor types, behavior
tests, and review prove how injected constructors propagate at runtime.
