# Review Lanes

Name review lanes for the workstream risk they cover. Avoid generic lanes such
as reviewer, explorer, or auditor unless the role is tightened by a concrete
concern and evidence base.

Useful lane patterns:

- setup mechanics;
- information shape;
- canonicality boundary;
- proof ledger;
- integration state;
- closure readiness;
- adversarial evidence honesty.

Every lane output should state evidence, confidence, finding disposition
proposal, and what it must not be used to prove.
