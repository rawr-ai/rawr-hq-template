## Implemented Boundary

C5 now has one `@rawr/agent-plugin-lifecycle` oRPC service with the exact `releases`, `vendors`, `packaging`, `exports`, `providers`, and `governance` module set. Consolidation did not collapse state authority: the immutable artifact repository, each package output, each export destination ledger, each provider home, each repository-governed record set, and the controller-owned undo capsule remain distinct owners connected through one-way ports. The CLI selects explicit identities, binds resource providers, invokes one typed service procedure, and projects the result.

The command ontology is closed and exact:

- Curated: `agent:plugins:attest-promotion`, `agent:plugins:build`, `agent:plugins:check`, `agent:plugins:create`, `agent:plugins:export`, `agent:plugins:package`, `agent:plugins:retire`, `agent:plugins:status`, `agent:plugins:sync`, `agent:plugins:test`, `agent:plugins:undo`, `agent:plugins:vendors:status`, and `agent:plugins:vendors:update`.

No C5 code mounts web plugins, composes apps, compiles runtime definitions, or repairs the retired app/plugin composition model. That exterior remains owned by the final-architecture migration cited above.
