# Runbook Quarantine

This directory contains runbooks that mix useful operational history with stale plugin, CLI, manifest, or runtime-composition guidance.

The runbooks are preserved intact because runbook intent is not deducible from architecture specs. They are not active instructions.

Use active runbooks from `docs/process/RUNBOOKS.md`. Revisit this directory after migration to decide which runbooks should be restored or rewritten against the implemented architecture.
