# System Evidence Maps

This directory groups evidence by runtime system or subsystem. These files are
operational maps: they explain what the evidence is attached to, why it matters,
what it impacts, and where the proof ceiling remains.

## Map

| Need | Open |
| --- | --- |
| Runtime-spine proof terrain across authoring, compiler, providers, hosts, async, telemetry, and handoff | `runtime-spine-evidence-map.md` |
| Effect-native integration categories and unresolved runtime-substrate questions | `effect-integration-map.md` |
| Telemetry, diagnostics, OTLP/HyperDX, catalog, and migration/control-plane observation evidence | `telemetry-observation-map.md` |

System maps are not proof authority. Promote or demote proof only through
`../proof-manifest.json`, source/fixture/test gates, and the structural/report
verification loop.
