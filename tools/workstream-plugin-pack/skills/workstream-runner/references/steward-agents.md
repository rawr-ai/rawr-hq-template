# Steward Agents

Steward agents are read-only helpers for workstream mechanics. They return
findings to the DRA; they do not decide that work is done.

Use the smallest steward set that changes the quality of the current
workstream.

## Opening Steward

Use before implementation when opening mechanics are non-trivial: objective,
boundary, authority order, stale inputs, prior assimilation, selected
capabilities, agent/wave packets, design lock, and stop conditions.

## Proof Ledger Auditor

Use when claim strength matters: proof/evidence categories, evidence homes,
waivers, deferred inventory, promotion boundaries, non-claims, and finding
disposition.

## Closure Steward

Use before final handoff when closure mechanics matter: output contract,
artifacts, checks, review disposition, scratch cleanup, repo/Graphite state,
deferred inventory, and zero-context Next Packet.

All stewards must stay read-only and must not convert subject-specific rules
into generic workstream law.
