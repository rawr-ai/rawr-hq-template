# API Server Plugin

`plugin-server-api` narrows the server-plugin kind to a caller-facing API
package. Its packet owns only the public client/API faces and embedded service
boundary that are additional to the reusable `service` kind. The API face
publishes server-side semantic operations; the application host owns Elysia
and oRPC fetch-transport realization. Habitat/Grit owns the installed
structural relations; native Nx dependency constraints remain a red
installation gap. The package remains package-less: its required
`tsconfig.json` owns TypeScript resolution without creating a package identity.
The closed shell permits a local `AGENTS.md`; the `agent-router` blueprint
alone owns its placement and source shape.
