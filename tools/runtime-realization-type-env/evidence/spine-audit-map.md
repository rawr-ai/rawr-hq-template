# Spine Audit Map

This map keeps the type environment honest about what it proves now and what it deliberately leaves unresolved.

| Issue | Harness status | Treatment |
| --- | --- | --- |
| Accepted descriptor refs | Proof | `ExecutionDescriptorRef` is boundary-discriminated and rejects impossible async owner shapes. |
| Accepted Effect authoring | Proof | RAWR `.effect(...)` is the runtime-realization authoring terminal; `.handler(...)`, Promise bodies, and raw generator yields fail type checks. This is not an oRPC-native API claim. |
| Accepted curated Effect public surface | Vendor proof | `@rawr/sdk/effect` is backed by real `effect@3.21.2` while curated authoring still rejects raw runtime constructors and unverified `Effect.pipe` spelling. |
| Accepted invocation clients | Simulation proof | The mini runtime resolves a registry boundary and supplies runtime-bound values through invocation context; service procedure calls still require `.withInvocation(...)`. |
| Accepted portable refs only | Proof | Portable artifacts reject descriptor tables and executable closures. |
| Accepted provider profile closure | Simulation proof | Selected providers must cover non-optional provider resource requirements before boot. |
| Effect runtime substrate vendor behavior | Vendor proof | Real Effect execution, `ManagedRuntime`, `Exit`/`Cause`, scope/finalizers, interruption, and composition spelling are tested against the installed package. |
| Process-local coordination vendor behavior | Vendor proof | Real `Queue`, `PubSub`, `Ref`, `Deferred`, `Schedule`, and `Stream` are tested as process-local primitives only. |
| Boundary vendor shapes | Vendor proof | TypeBox, oRPC, and Inngest probes prove narrow package shapes without claiming production host semantics. |
| Middle-spine derivation and compiler | Simulation proof | Explicit lab declarations, including explicit async descriptors and cold server route factories, derive normalized graph artifacts, descriptor refs/table inputs, service binding plans, surface plans, server route descriptors, dispatcher descriptor inventory, explicit dispatcher operation inventory, async owner-to-step membership artifacts, refs-only portable artifacts, compiled process plan, provider graph, registry input, native-shaped adapter lowering payloads, harness placeholders, diagnostics, and scoped bootgraph input without executing descriptor bodies. This does not prove production SDK extraction, final route import-safety law, final async membership authoring syntax, final dispatcher access policy, production provider lowering, or real harness mounting. |
| Bootgraph/catalog/finalization | Simulation proof | Fake boot modules prove dependency ordering, startup rollback, reverse finalization that continues after finalizer failures, and redacted in-memory lifecycle/catalog records. Provider provisioning modules additionally lower lab-internal acquire/release hooks through graph-driven dependency-ordered mini bootgraph execution with graph diagnostic fail-closed behavior, rollback, reverse finalization, release-failure recording, and redacted catalog records without deciding final public `ProviderEffectPlan` shape. |
| Runtime access and service binding cache | Simulation proof | The mini runtime exposes sanctioned resource/optional resource/telemetry/diagnostic/topology access, redacts observation records, validates explicit service binding dependency inputs before cache construction, rejects missing, ambiguous, and cyclic dependency refs, materializes dependencies before dependents, structurally encodes service binding cache keys, and lazily reuses service bindings by construction-time inputs while excluding invocation from cache identity. Final method law and production service binding compiler topology remain open. |
| Mini runtime registry/invocation | Simulation proof | Descriptor table, registry, full ref identity checks, runtime-owned Effect access, adapter payload delegation, and deployment handoff are tested in the contained miniature. |
| P0 async step membership | Expected fail | Explicit lab async step refs now prove owner-to-step membership artifacts and one-owner diagnostics, but final public workflow/schedule/consumer membership metadata syntax remains unresolved. |
| P1 ProviderEffectPlan shape | Expected fail | The plan is intentionally opaque until the spec locks the producer/consumer fields. |
| P1 Effect managed runtime substrate | Expected fail | Vendor primitives are now proven, but the final RAWR-owned substrate public/internal contract remains open. |
| P1 process-local coordination resources | Expected fail | Vendor primitives are now proven as local infrastructure, but final RAWR resource contracts and method law remain open. |
| P1 provider plan lowering | Simulation proof | Lab-internal provider acquire/release hooks lower through mini bootgraph/provisioning with real Effect execution, provider graph diagnostic fail-closed behavior, graph-driven dependency ordering, dependency value flow, rollback/release on failed acquire, reverse finalization, release-failure recording, and redacted catalog records. Final public plan shape and production bootgraph integration remain open. |
| P1 Effect boundary policy matrix | Expected fail | Timeout, retry, interruption, telemetry, redaction, and error/exit mapping need locked metadata before positive proof. |
| P1 safe Effect composition surface | Expected fail | The curated helper list and vendor-aligned names remain open; raw runtime constructors stay forbidden. |
| P1 dispatcher access | Expected fail | Explicit lab dispatcher descriptors now preserve operation inventory and reject unlisted workflow operations, but the type env does not decide `useWorkflowDispatcher(...)`, explicit-vs-ambient policy, or final public SDK/DX shape. |
| P1 RuntimeResourceAccess | Expected fail | The mini-runtime facade now proves a narrow sanctioned method set, but final method law is intentionally not invented. |
| P2 server route derivation | Simulation proof plus expected fail | Lab-local cold route factories derive server route descriptors, server execution refs, descriptor table inputs, surface runtime plans, and refs-only portable artifacts without executing route bodies; failed factories, invalid route paths, unsafe import markers, duplicates, and explicit server refs without route-factory coverage stay diagnostic. Final public route import-safety law, native oRPC/Elysia lowering, OpenAPI publication, and real HTTP paths remain unresolved. |
| P2 adapter Effect callback lowering | Simulation proof plus expected fail | Lab-local native-shaped server callback payloads now consume pre-derived route descriptors/refs, reject wrong-boundary or mismatched route metadata, preserve full ref identity, avoid executable descriptors in payloads, and delegate through `ProcessExecutionRuntime`; production Elysia/oRPC callback lifecycle and final boundary policy remain open. |
| P2 async Effect bridge lowering | Simulation proof plus expected fail | Lab-local async bridge payloads now consume explicit owner-to-step refs, reject invalid or mismatched owner metadata, preserve full ref identity, avoid executable descriptors in payloads, and delegate through `ProcessExecutionRuntime`; native Inngest FunctionBundle/worker mounting, durable scheduling/retry/idempotency, and final async host policy remain open. |
| P2 first server/async harness mounts | Simulation proof plus expected fail | Lab-local server and async started harnesses now consume only adapter lowering payloads, expose route/step callbacks, preserve full ref identity, record start/invoke/stop lifecycle events, reject raw descriptors/compiler plans, and delegate through `ProcessExecutionRuntime`; production Elysia/oRPC/Inngest mounting, real HTTP/worker paths, other harness kinds, durable async semantics, deployment wiring, and final boundary policy remain open. |
| P2 initial resource/provider cut | TODO | Catalog candidates are planning input, not canonical ids. |
| P2 runtime profile config redaction | Simulation proof | Lab-contained provider provisioning validates config through `RuntimeSchema`, fails closed before provider build/acquire on invalid config, records diagnostic-safe validation failures and redacted config snapshots, and keeps secrets/live handles out of catalog and provisioning trace records. Production config precedence, platform secret stores, telemetry export, and catalog persistence remain open. |
| Source-hygiene drift | Out of scope | The stale repo-local spec copy must be handled elsewhere before migration planning. |
